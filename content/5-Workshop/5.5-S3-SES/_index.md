Next, we replace two important components that were running locally with managed AWS services. **Amazon Simple Storage Service (Amazon S3)** stores files such as images, videos, and certificates, while **Amazon Simple Email Service (Amazon SES)** handles outbound email instead of Gmail SMTP.

Using managed services makes the application easier to scale, more reliable, and less dependent on EC2 instance resources.

#### Amazon S3 — Object storage

Create separate S3 buckets for each data type. All buckets are configured as **Private** with **Block Public Access** enabled so only the application or authorized services can access them.

| Bucket | Purpose |
|--------|---------|
| `dacswebsk-images-quan` | Event images |
| `dacswebsk-videos-quan` | Learning videos |
| `dacswebsk-certificates-quan` | Certificates generated by the system |
| `dacswebsk-uploads-quan` | Assignments and supporting files |

After creating the buckets, review the list to confirm they were provisioned successfully.

![S3 buckets list](/images/5-Workshop/5.5-S3-SES/09-s3-buckets-list.png)

#### Verify file upload

After S3 is configured, create a new event in the admin UI and upload a cover image. If configuration is correct, the file is stored in the corresponding bucket under the `events/` prefix.

![Event objects on S3](/images/5-Workshop/5.5-S3-SES/10-s3-events-objects-uploaded.png)

The ASP.NET Core application uses **`IFileStorageService`** with an **S3 provider** to manage uploads and storage. Files are no longer saved on the EC2 disk; they are stored as objects in Amazon S3, improving durability, scalability, and reducing data loss risk when replacing or rebuilding the server.

#### Amazon SES — Email delivery

Configure **Amazon Simple Email Service (Amazon SES)** to replace Gmail SMTP.

Follow these steps:

1. Open **Amazon SES → Verified identities**.
2. Verify the sender email address (for example: `testmailxn@gmail.com`).
3. Confirm the email identity status is **Verified**.
4. Send a test email from the AWS Management Console or trigger email from the application.

![SES identity Verified](/images/5-Workshop/5.5-S3-SES/11-ses-identity-verified.png)

> **Note:** New AWS accounts typically start with Amazon SES in **Sandbox** mode. In this mode, email can only be sent to verified addresses. For production, request **Production Access** to remove this limitation.

#### Section summary

In this section, you integrated **Amazon S3** and **Amazon SES** into the Event Management system. Amazon S3 replaces local file storage under `wwwroot`, providing durable and scalable object storage. Amazon SES replaces Gmail SMTP for application email. Together, these services remove local-environment dependencies and move the system closer to a cloud-native architecture on AWS.
